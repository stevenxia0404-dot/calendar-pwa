### v1.1 迭代（2026-07-17）

#### 每日 Task 功能
- 新增 `type` 字段区分 `event`（定时日程）和 `task`（当日任务）
- 新增 `completed` 字段标记任务完成状态
- **自动分类**：输入无日期/时间关键词 → 自动归为当日 task，`time` 为空
- **Task UI**：左侧圆形 checkbox，点击切换完成 → 标题加 `line-through` + 变灰，不删除
- 排序：未完成在上、已完成在下，方便月底绩效回顾
- **iCal 导出**：task 以全天事件（`VALUE=DATE`）导出，完成的加 `✓` 前缀

#### API 变更
- `POST/PUT/GET /events` 支持 `type` + `completed` 字段
- 新增 `PUT /events/:id/toggle` 轻量切换完成状态
- `POST /events` 不再强制要求 `time`（task 可为空）
- DB 迁移 `004_tasks.sql`：events 表加 `type` 和 `completed` 列

#### 同步策略优化
- **全量同步改为合并逻辑**：不再粗暴覆盖本地数据，改为逐条比较 `updated_at`
- 修复：本地新创建的 task 在推送到云端前不会被全量同步覆盖消失

#### 布局重构
- 日历和日程详情**左右并排**（Desktop：左侧 1/4 日程，右侧 3/4 日历）
- 内容区填满 header 到输入栏之间的可用高度
- 无详情卡片时日历自动撑满全宽
- Mobile 保持上下堆叠（`md:` 断点）

#### Bug 修复
- **Hydration 错误修复**：`speechSupported` 从直接判定改为 `useState(false)` + `useEffect` 客户端激活，避免 SSR/CSR 渲染不一致

#### 小组件模式（已回退）
- 尝试添加半透明磨砂玻璃小组件模式 → 无实质场景价值，已移除
- PWA 桌面小组件不具备可行性（无法置顶、无法嵌入桌面），仅保留讨论结论

## v0.4.0 重构记录（2026-07-14）

### 决策背景
v0.3.0 卡在部署阶段 3 个月，核心问题复盘：
- **GitHub OAuth** 不适合国内用户（无 GitHub 账号）
- **Coze AI** 增加延迟和成本，90%+ 场景前端规则引擎已覆盖
- **语音输入** 是假的 2 秒动画
- **同步策略** 云端覆盖本地，跨设备编辑会丢数据
- Coze Token 硬编码在源码中，已泄露到 Git 历史

### 改动内容

**前端（src/app/page.tsx）**
- 砍掉 Coze AI 工作流及 `COZE_CONFIG`
- 新增浏览器原生语音识别：按住说话、松开发送、滑动取消（56px 大按钮）
- Safari 不渲染麦克风按钮，用 iOS 键盘自带听写
- GitHub OAuth 替换为邮箱验证码激活（弹窗交互，无感登录）
- 日程 ID 从 Date.now() 换成 crypto.randomUUID()
- 新增 updatedAt 字段，同步策略改为时间戳合并

**后端（workers-api/src/index.js）**
- GitHub OAuth 相关代码全部移除
- 新增邮箱验证码认证：/auth/send-code、/auth/verify、/auth/me、/auth/account
- JWT 修复：btoa 改为标准 base64url 编码，新增 exp 过期
- 新增 MailChannels 免费发邮件
- 同步 API 增强：GET /events?since= 增量拉取、POST /events/merge 合并
- 所有写入支持 INSERT OR REPLACE + updated_at 比较

**配置（workers-api/wrangler.toml）**
- 移除 GitHub OAuth 相关变量
- 新增 SENDER_EMAIL，JWT_SECRET 改为 secret 管理

### v1.0 正式版（2026-07-15）

#### 时间解析完善
- 支持中文数字时间：「下午三点」→ 15:00、「十点半」→ 10:30、「两刻」→ 30 分
- 修复「两」丢失、「分」正则太窄、刻优先于两的判断顺序
- 支持换行分隔批量创建：每行一个日程，一次回车全部保存

#### iCal 日历订阅修复
- UID 固定为数据库 id，不再每次刷新变化导致苹果日历同步异常
