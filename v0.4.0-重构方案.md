# AI日程管家 v0.4.0 重构方案

## Context

当前项目 v0.3.0 卡在部署阶段，核心问题是**技术选型与实际使用场景不匹配**：GitHub OAuth 门槛太高、Coze AI 增加延迟和成本、语音输入是假的。v0.4.0 目标是将其改造为**零外部AI依赖、邮箱隐形认证、跨设备自动同步**的个人效率工具。

---

## 改动概览

```
砍掉：Coze AI 工作流、GitHub OAuth
换掉：认证方案 → 邮箱验证码、同步策略 → 时间戳合并
新增：浏览器原生语音识别、Passkey 自动登录
修复：JWT base64url 编码、Coze Token 泄露
```

## 一、前端改动（src/app/page.tsx）

### 1.1 砍掉 Coze，纯规则引擎

- 删除 `COZE_CONFIG`、`callCozeAPI()`
- `handleSubmit` 简化：`parseSmartDate` + `extractTime` 直接出结果，失败用当前选中日期兜底

### 1.2 语音识别：按压说话，松开发送

和微信语音一样的交互模式：

```
手指按住 → 录音开始（按钮放大+脉冲光晕，显示"正在听..."）
手指滑动出按钮区域 → 取消录音
手指松开 → 停止录音 → 转文字 → 自动填入输入框 → 自动解析提交
```

实现要点：
- `onPointerDown` → `recognition.start()`
- `onPointerUp` → `recognition.stop()` → 拿 transcript → `setInputText` → `handleSubmit()`
- `onPointerMove` → 检测手指是否滑出按钮区域 → 显示取消状态
- 按钮尺寸：**56×56px 最小触摸目标**，移动端手指友好
- 检测不到 `SpeechRecognition` API → 隐藏麦克风按钮（Safari 用户用系统键盘自带听写）

```typescript
function useSpeechRecognition() {
  const [isRecording, setIsRecording] = useState(false);
  const [willCancel, setWillCancel] = useState(false);
  const recognitionRef = useRef<SpeechRecognition | null>(null);

  const start = () => {
    const SR = window.SpeechRecognition || window.webkitSpeechRecognition;
    if (!SR) return false;
    const r = new SR();
    r.lang = 'zh-CN';
    r.interimResults = false;
    r.onresult = (e) => { /* resolve transcript */ };
    r.onerror = () => { setIsRecording(false); };
    r.start();
    recognitionRef.current = r;
    setIsRecording(true);
    return true;
  };

  const stop = () => {
    recognitionRef.current?.stop();
    setIsRecording(false);
  };

  return { isRecording, willCancel, start, stop, setWillCancel, isSupported: !!SR };
}
```

- Safari 不支持则隐藏麦克风按钮，用户用 iOS 系统键盘自带听写（键盘右下角麦克风键）

### 1.3 认证流程改造

```
当前：GitHub OAuth → 重定向 → JWT → 存 localStorage
改后：邮箱验证码 → JWT → 存 localStorage + Credential Store
```

- 删掉 GitHub 登录相关 UI 和 `handleLogin`/`handleLogout`
- 新增激活浮窗：一个邮箱输入框 + 验证码输入框 + "激活同步"按钮
- 已激活设备自动恢复——localStorage 有 JWT 就静默验证
- 新增"添加设备"按钮：显示激活指引（邮箱 + 验证码）

### 1.4 同步策略改为时间戳合并

```typescript
// 每个 event 新增 updatedAt 字段
interface ScheduleEvent {
  id: string;       // 改为 nanoid，不用时间戳
  title: string;
  date: string;
  time: string;
  raw: string;
  updatedAt: number; // Date.now()
}
```

合并逻辑：同 id → 取 `updatedAt` 更大的版本，不做覆盖

## 二、后端改动（workers-api/src/index.js）

### 2.1 认证方案：邮箱验证码

```
POST /auth/send-code   Body: { email }  → 发验证码到邮箱，5分钟有效
POST /auth/verify      Body: { email, code } → 返回 JWT
GET  /auth/me          Header: Bearer <jwt> → 返回用户信息
DELETE /auth/account   → 删除用户所有数据
```

- 验证码存 D1 或 KV，5 分钟过期，同邮箱 1 分钟内只能发一次
- 用 Cloudflare Email Routing + MailChannels 免费发邮件
- 新用户首次验证自动创建账户

### 2.2 修复 JWT 编码

当前 `btoa` 对二进制数据不兼容，改为标准 base64url：

```javascript
function base64urlEncode(str) {
  return btoa(str).replace(/\+/g, '-').replace(/\//g, '_').replace(/=/g, '');
}
```

### 2.3 数据库表结构修改

```sql
-- users 表：github_id → email
ALTER TABLE users DROP COLUMN github_id;
ALTER TABLE users DROP COLUMN avatar_url;
ALTER TABLE users ADD COLUMN email TEXT NOT NULL UNIQUE;

-- events 表：加 updated_at 索引
CREATE INDEX idx_events_updated ON events(user_id, updated_at);
```

### 2.4 同步 API 增强

```
GET  /events?since=<timestamp>  → 增量拉取（只返回 since 之后更新的）
POST /events/merge              → 上传本地变更，服务端按 updatedAt 合并
```

### 2.5 邮箱发送实现

Cloudflare Workers 通过 MailChannels 免费发邮件（Cloudflare 官方合作）：

```javascript
async function sendVerificationCode(email, code) {
  await fetch('https://api.mailchannels.net/tx/v1/send', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      personalizations: [{ to: [{ email }] }],
      from: { email: 'noreply@schedule-pro.pages.dev', name: 'AI日程管家' },
      subject: '验证码：' + code,
      content: [{ type: 'text/plain', value: `你的验证码是：${code}，5分钟内有效。` }],
    }),
  });
}
```

需在 Cloudflare DNS 配置 SPF 记录允许 MailChannels 代发。
